# このプロジェクトの決まりごと

## コミットメッセージは日本語で書く

毎日 21:00 に、その日のコミットが Discord に自動で投稿されます
(`.github/workflows/daily-discord-report.yml`)。

**このレポートは子供が読みます。** コミットの件名がそのまま流れるので、
専門用語を避けた日本語で書いてください。

### 書き方

1行目は「**何がどう変わったか、遊ぶ人から見てどうなるか**」を短く。

```
ブロックの絵をきれいにした
```

技術的な詳細が必要なら、1行空けて本文に書く。本文は英語でも日本語でも構いません
(レポートには件名しか出ません)。

```
ブロックの絵をきれいにした

ホットバーとインベントリの見た目が単色の四角だったので、
実際に描画で使っているテクスチャアトラスから切り出すようにした。
石と鉱石が見分けられるようになる。
```

### 良い例

```
草原の色を青にした
水から陸にあがれるようにした
持っているブロックが画面に出るようにした
道具がないと石が取れない理由を出すようにした
遠くの景色を消して動きを軽くした
クラフトで道具を作れるようにした
```

### 避けたい例

```
Draw block icons from the texture atlas   ← 英語
Fix #42                                    ← 何が直ったのか分からない
refactor                                   ← 遊ぶ人には意味が伝わらない
main.js を修正                              ← ファイル名だけでは何が変わったか不明
```

### 迷ったときの目安

「**これを読んだ子供が、ゲームのどこを見に行けばいいか分かるか**」で判断してください。
`chunk` `mesh` `refactor` `API` のような言葉が出てきたら、言い換えられないか考える。

日次レポートは直近24時間ぶんしか拾わないため、そのまま残して構いません。
