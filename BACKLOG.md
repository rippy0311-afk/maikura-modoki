# 今後のアップデート候補

作業中に見つかった未解決の課題。着手したら該当項目を消すか、コミットハッシュを添えて履歴に残す。
確認日: 2026-07-20

---

## 優先度: 中

### 1. 装備に効果がない

`iron_helmet` などはクラフトでき所持数も持てるようになったが、**装備しても被ダメージは変わらない**。
`damagePlayer()` は防具を一切参照していない。
装備スロットへの装着操作も未実装で、`armor-grid` はクリックしても何も起きない。

### 2. クラフトが簡易版

現在は「材料が揃っていればボタンで作れる」形式。
2x2 グリッドに実際にアイテムを置く本格版にするなら、レシピの形状定義とドラッグ&ドロップの実装が必要。
作業台の 3x3 グリッドを入れる場合はさらに拡張が必要。

---

## 優先度: 低

### 3. 未使用の定義が残っている

いずれも定義だけで参照が1箇所のみ:

- `DRAGON_KIT_CATEGORIES` — 7カテゴリ40アイテム超の装備セット定義
- `CUSTOM_BREAK_BLOCKS` — 破壊可能ブロックの一覧
- `getDragonKitItem()` / `getSelectedCustomItem()` — `return null` のスタブ

作りかけの機能の残骸。使う予定が無いなら消すと `main.js` が読みやすくなる。

### 4. 所持数のキー空間が混在している

`survivalInventory` は「ブロックなら数値のブロックID、それ以外は文字列ID」でキーを持つ。
JS のオブジェクトキーは常に文字列なので動作上は問題ないが、読み手には分かりにくい。
全アイテムを文字列IDに統一する余地がある。

### 5. main.js が大きい

レンダラ初期化・入力・UI・インベントリ・サバイバル・セーブ・メインループが `main.js` に同居している。
クラフトは `js/crafting.js` に分離済みなので、同様に UI 層とセーブ層を切り出せる。

### 6. auto-push が Windows 専用

`scripts/auto-push.ps1` は PowerShell + `System.IO.FileSystemWatcher` 依存で、macOS / Linux では動かない。
共同編集者が別 OS の場合は手動運用になる。

### 7. テストハーネスが残っていない

WebGL をスタブして `main.js` をそのまま動かし、破壊・取得・クラフト・UI表示を検証する仕組みを一時的に作ったが、リポジトリには入れていない。
これがあると「アイテムが取れない」類の問題を数分で切り分けられる。

### 8. アイコンをさらに良くする

現在はテクスチャアトラスから側面タイルを切り出して使用している。
これ以上を求めるなら、CC0 素材の導入か、`blocks.js` の `makeAtlasCanvas()` を拡充する。

Minecraft のアセットは使えない。
本リポジトリは public かつ GitHub Pages で公開配信しているため、アセットを入れた時点で再配布に当たり、Mojang の EULA に抵触する。
