# 今後のアップデート候補

作業中に見つかった未解決の課題。着手したら該当項目を消すか、コミットハッシュを添えて履歴に残す。
確認日: 2026-07-18 (`dffcd4b` 時点)

---

## 優先度: 高

### 1. ブロックの編集内容がセーブされない

`makeCurrentState()` が保存するのは以下だけ:

```
version worldId preset seed mode hp fly blockColors blockTextures x y z yaw pitch savedAt
```

ボクセルデータ (`world.data`) が含まれていない。ワールドは preset + seed から再生成されるため地形は完全に再現されるが、**プレイヤーが建てたもの・掘った穴は再訪時にすべて消える**。

サンドボックスとして最大の欠落。素直に全量 (480x384x480 = 84MB) を持つのは非現実的なので、
「生成結果との差分だけを持つ」方式が要検討。`World` の編集を差分 Map に記録すれば、
セーブ量は実際に触ったブロック数に比例する。

### 2. 起動時に約2.4秒フリーズする

`generateWorld` 約0.9秒 + 全900チャンクのメッシュ化 約1.5秒。
進捗バーは出るようになったが、メインスレッドを占有する構造自体は変わっていない。

未実施の対策:

- **遅延メッシュ化** — 起動時はプレイヤー周辺だけメッシュ化し、残りは `requestIdleCallback` で分散。体感の固まりが消える
- **ワールド縮小** — 480³ → 256³ など。メモリ84MBも同時に減るが、世界が狭くなる

済んでいる対策: `columnTop` による打ち切りと添字直接参照 (`c0c84eb`、地形により13〜28%短縮)、
描画距離カリング + 設定UI (`c0c84eb`)。

### 3. チャットが通信していない

`chatState.roomId` を持つがネットワーク層が無く、自分の発言をローカル表示するだけ。
WebRTC ボイスチャット (`js/voice.js`) は入ったので、テキストも同じ経路に載せられる可能性がある。

**先に直すべき点**: `renderChat()` が `main.js:2125` で生テキストを `innerHTML` に入れている。
ネットワーク化すると他人の入力がそのまま HTML として実行される。`textContent` に変えるか
エスケープが必要。同種の箇所がワールド名 (`main.js:544`) にもある。

`index.html` の `<script>` に含まれておらず、完全に死んでいる:

| ファイル | 内容 |

| `js/image.js` | 画像をボクセルに変換して配置 |

| `js/minecraft_catalog.js` | ブロック/アイテムのカタログ定義 |

実害あり: `main.js` の `getCatalogItem()` は `MINECRAFT_GENERAL_CATALOG` を参照するが、
`minecraft_catalog.js` が読み込まれていないため常に `null` を返す。
クリエイティブでカタログアイテムを設置する処理が無言で機能しない。

**復活させるか削除するかを決める**。`_test.html` は landmarks と image を読み込んでいるので、
そちらが動作確認用として残っている。

### 5. 文字化け

Shift-JIS 環境から混入したと思われる `?` 潰れ。

- `README.md` — 17行が `????` 状態。ほぼ全文が読めない

### 6. 装備に効果がない

`iron_helmet` などはクラフトでき所持数も持てるようになった (`f1a2ef5`) が、

ドラッグ&ドロップの実装が必要。作業台 (3x3) を入れるならさらに拡張。

## 優先度: 低

### 8. 未使用の定義が残っている

いずれも定義だけで参照が1箇所 (自身の定義) のみ:

- `DRAGON_KIT_CATEGORIES` — 7カテゴリ40アイテム超の装備セット定義
- `CUSTOM_BREAK_BLOCKS` — 破壊可能ブロックの一覧
- `getDragonKitItem()` / `getSelectedCustomItem()` — `return null` のスタブ

作りかけの機能の残骸。使う予定が無いなら消すと `main.js` が読みやすくなる。

### 9. 所持数のキー空間が混在している

`survivalInventory` は「ブロックなら数値のブロックID、それ以外は文字列ID」でキーを持つ。
JS のオブジェクトキーは常に文字列なので動作上は問題なく、`getInventoryCount()` で
両方引けるが、読み手には分かりにくい。全アイテムを文字列IDに統一する余地がある。

移行しても既存セーブは壊れない (そもそもインベントリを保存していないため)。

### 10. main.js が 2363 行

全体 4419 行のうち半分以上。レンダラ初期化・入力・UI・インベントリ・サバイバル・
セーブ・メインループが1ファイルに同居している。クラフトは `js/crafting.js` に分離済み (`f1a2ef5`)。
同様に UI 層とセーブ層を切り出せる。

### 11. auto-push が Windows 専用

`scripts/auto-push.ps1` は PowerShell + `System.IO.FileSystemWatcher` 依存で、
macOS / Linux では動かない。共同編集者が別 OS の場合は手動運用になる。

### 12. テストハーネスが残っていない

WebGL をスタブして `main.js` をそのまま動かし、破壊・取得・クラフト・UI表示を
検証する仕組みを一時的に作ったが、リポジトリには入れていない。

これがあると「アイテムが取れない」類の問題を数分で切り分けられる。実際、
ホットバーが空のまま埋まらない問題 (`453360f`) と、クリエイティブとサバイバルの
挙動差 (`52acdb4`) はこの方法で特定した。常設するか要判断。

### 13. アイコンをさらに良くする

現在はテクスチャアトラスから側面タイルを切り出して使用 (`dffcd4b`)。
これ以上を求めるなら:

- **CC0 素材の導入** — Kenney.nl のボクセル素材などはパブリックドメインで、公開リポジトリでも安全
- **手続き生成の作り込み** — `blocks.js` の `makeAtlasCanvas()` を拡充する。ライセンス問題なし

**Minecraft のアセットは使えない**。本リポジトリは public かつ GitHub Pages で公開配信して
いるため、アセットを入れた時点で再配布に当たり、Mojang の EULA に抵触する。
